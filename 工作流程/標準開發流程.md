## 🎯 標準開發流程

#### 1.1 需求分析
- [ ] 明確專案目標
- [ ] 列出核心功能
- [ ] 確定技術棧
- [ ] 評估時間和資源

#### 1.2 架構設計
- [ ] 設計系統架構
- [ ] 規劃資料庫 schema
- [ ] 設計 API 介面
- [ ] 建立 implementation_plan.md

#### 1.3 任務拆分
- [ ] 建立 task.md
- [ ] 設定里程碑
- [ ] 評估風險

### 階段 2: 開發 (EXECUTION)

#### 2.1 環境設定
- [ ] 初始化專案
- [ ] 安裝依賴
- [ ] 設定開發環境
- [ ] 配置版本控制

#### 2.2 核心開發
- [ ] 按照 task.md 順序開發
- [ ] 遵循程式碼規範

- [ ] 撰寫註解和文檔

#### 2.3 問題解決
- [ ] 清楚描述問題
- [ ] 提供錯誤訊息

- [ ] 記錄解決過程

#### 3.1 測試
- [ ] 功能測試
- [ ] 邊界測試
- [ ] 效能測試

#### 3.2 優化
- [ ] 程式碼重構
- [ ] 效能優化
- [ ] UI/UX 調整

#### 3.3 驗收
- [ ] 建立 walkthrough.md
- [ ] 記錄測試結果
- [ ] 準備部署

## 💡 最佳實踐

### 與 AI 協作
1. **清楚表達**: 明確說明需求和期望
2. **提供上下文**: 附上相關程式碼和錯誤訊息
3. **快速迭代**: 及時測試並回饋結果
4. **記錄決策**: 重要決策寫入文檔

### 程式碼品質
1. **模組化**: 功能拆分,單一職責
2. **可讀性**: 清晰的命名和註解
3. **可維護性**: 避免過度複雜
4. **可測試性**: 易於測試的設計

### 時間管理
1. **番茄工作法**: 25 分鐘專注 + 5 分鐘休息
2. **時間記錄**: 記錄每日工作時間
3. **定期回顧**: 每週回顧進度
4. **作息平衡**: 避免過度加班
