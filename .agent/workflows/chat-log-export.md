# 對話紀錄匯出與精準審計流程 (Chat Log Export Workflow)

為了消除「無檔案變動期」的工時計算盲點（例如長達 4 小時的測試或思考期），請遵循以下流程將對話紀錄導入 CPDM 系統。

## 1. 執行時機
* 進行了長達 1 小時以上且涉及大量「手動測試」或「邏輯討論」的對話。
* 對話中沒有產生大量的 .md 文件或 Artifacts。
* 您希望該段時間被 100% 精確記錄，而非透過系統「戰略性休息」公式校正。

## 2. 操作步驟
1. **導出日誌**：在 Antigravity 對話介面，點擊「導出對話紀錄」或「下載 Markdown」。
2. **存放路徑**：將下載的 `.md` 檔案存放到專案目錄：
   `c:\Users\Arthur8888tw\Documents\Html\新增資料夾\AI專案管理\chat_logs\`
3. **重新命名**：為了讓系統自動對齊，請將檔案重新命名為該對話的 **ID**：
   * 範例：`9caf95ae-a3c8-49ba-b4a9-3566bf990c80.md`
   * (對話 ID 可以在儀表板的「詳細記錄列表」或對話網址中找到)

## 3. 系統連動
* **自動分析**：腳本 `generate_interaction_history.js` 會優先檢查 `chat_logs` 目錄。
* **高精確度**：系統將自動解析 Markdown 內的訊息時間戳記（Timestamp），將對話中的每一分一秒都視為「活動狀態」。
* **日誌摘要**：系統會自動從 Log 中提取關鍵對話內容填入「摘要 (Summary)」欄位。

## 4. 目錄維護
* 定期清理過舊的 `chat_logs` 檔案。
* 若已在 `conversation_metadata.json` 手動鎖定 (locked: true) 該工時，則不需再放置 MD 檔案。
