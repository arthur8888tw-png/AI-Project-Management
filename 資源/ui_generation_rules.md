# AI UI 助手生成規範 (AI UI Generation Rules)

為降低後續 UI 調整的工時（目前已佔 40%），本規範定義了 AI 在產生畫面時必須執行的「準則」。

## 1. 字體與排版 (Typography & Hierarchy)
- **拒絕極小字**：除非特殊說明，禁止使用 `text-[10px]` 或更小的字。基礎標籤 (Label) 至少為 `text-xs` (12px)，正文為 `text-sm` (14px)。
- **數據強化**：關鍵數值必須使用 `font-bold` 或 `font-black`，並在視覺上與背景有足夠反差（使用 Badge 或陰影）。
- **避重就輕**：導航與選單文字應維持 `text-base` (16px)，避免因字體過大造成的「壓迫感」。

## 2. 佈局與空間 (Layout & Spacing)
- **橫排優先**：在展現數據項目時，優先使用 `flex-row` 橫向佈局以節省垂直空間，並符合閱讀直覺。
- **對比性側欄**：比較圖表（如圓餅圖與清單）必須左右對開，而非上下疊放。
- **容器一致性**：使用 `rounded-2xl` 作為模塊化容器的標準圓角，配合 `shadow-sm` 與 `border-slate-200` 建立階層。

## 3. 色彩與一致性 (Color & Identity)
- **類別索引化**：特定類別（如「DEBUG」、「UI 調整」）的顏色必須在該頁面所有圖表與清單中**完全同步**。
- **情緒配色**：
    - 常態/正確：`Emerald-500`
    - 警告/偏差：`Rose-500`
    - 進程/處理：`Amber-500` 或 `Indigo-500`

## 4. 互動設計 (Interaction Patterns)
- **微互動**：所有按鈕或互動卡片必須具備 `hover:scale-[1.02]` 或 `hover:border-slate-300` 的反饋。
- **過渡動畫**：動畫時間建議在 `300ms` ~ `500ms` 之間，追求物理行為的「真實感」（如捲軸展開必須有彈性感）。

## 5. 無塵室原則 (Cleanliness)
- **移除捲軸**：全屏應用應確保 `overflow-hidden`，僅內容區可滾動並隱藏原生捲軸 (`no-scrollbar`)。
- **組件內封裝**：UI 變數（如顏色清單）應統一提取至腳本頂部，方便後續全局調整。
## 進階 AI UI 生成規則 (30 大範例準則)

為了確保 AI 生成的代碼能一次精確到位，請要求 AI 在執行時嚴格遵守這 30 條「大腦指令集」：

### 1. 角色與邏輯基礎 (Role & Logic Basis)
1.  **架構師角色**：要求 AI 以「資深前端架構師」身份開發，專精於現代、簡約的高級儀表板風設計。
2.  **結果導向指令**：定義「目的」而非「手法」（例如：描述「加強閱讀對比」而非「把字變黑」）。
3.  **物理連貫性校驗**：繪製數據前必須檢查邏輯（如：單日工時累計不得超過 24 小時）。
4.  **數據排序優先**：所有動態清單必須自動按時間（降序）或重要性排序，除非另有說明。
5.  **空狀態設計系統 (Empty States)**：所有數據驅動組件必須定義「暫無數據」的視覺展現，不得留白。
6.  **錯誤回饋機制**：若 JSON 數據格式異常或缺少欄位，必須在 UI 顯示具體的錯誤標籤或佔位提示。

### 2. 佈局與響應式開發 (Layout & Responsiveness)
7.  **全棧橫向佈局**：在數據細節項中優先使用 `flex-row`。避免不必要的上下堆疊以預防視覺壓迫感。
8.  **對稱佈局與對開**：圓餅圖/清單、統計卡/趨勢圖應左右對開（Side-by-side），最大化水平空間利用率。
9.  **絕對定位禁令**：除 Tooltip、浮動按鈕或 Modal 外，嚴禁使用 `absolute`。優先使用 Flexbox 或 Grid 維持佈局穩定性。
10. **響應式源頭注入**：所有容器必須包含 `md:` 以上的佈局定義，確保跨平台（手機/電腦）一次到位。
11. **黏性導航 (Sticky Positioning)**：大型資料表表頭必須使用 `sticky top-0`；長頁面的操作按鈕應固定於底部或頂部。
12. **垂直韻律與間距系統**：同一層級的模組區塊間距應固定（如 `mb-6` 或 `mb-8`），建立穩定的視覺節奏感。

### 3. 視覺深度與設計美學 (Visual Depth & Aesthetics)
13. **高反差 Badge 系統**：關鍵指標百分比需使用「高對比色塊標籤」包裹，移除透明度，強化數據可見度。
14. **雙重細節圓角**：模組大容器統一使用 `rounded-2xl`，內層小項目（如按鈕、輸入框）統一使用 `rounded-lg`。
15. **層次感投影 (Shadow Elevation)**：使用 `shadow-sm` 於容器，`hover:shadow-md` 於交互项，模擬真實物理層次。
16. **毛玻璃效果 (Glassmorphism)**：高級背景應應用 `backdrop-blur-md` 搭配 `bg-white/10`，創造 Premium 質感。
17. **微反饋交互**：所有可點擊項目必須具備 `hover:scale-[1.02]` 與平滑的 `border-color` 過渡動效。
18. **情緒色彩映射**：正向數據綠色 (`emerald-500`)，警告數據紅色 (`rose-500`)，處理中黃色 (`amber-500`)。

### 4. 字體與資訊密度 (Typography & Density)
19. **拒絕極小字體 (Min 12px)**：除了極少數標籤外，標籤禁止使用 `text-[10px]`。基礎讀取文字至少需為 `text-xs`。
20. **字體粗細分層**：數據標籤使用 `font-medium`，關鍵數值使用 `font-black`，導航文字使用 `font-bold`。
21. **字距優化 (Tracking)**：大標題或關鍵數值需使用 `tracking-tight`，提昇文字的沉穩感與專業度。
22. **文字保護機制**：長摘要必須搭配 `truncate` 或 `line-clamp` 限制行數，避免因內容過多撐破佈局。

### 5. 代碼品質與維護性 (Code Quality & Assets)
23. **CSS 變數封裝**：顏色與常用配置（如主題色、圓角值）必須提取為 CSS Variables 或腳本頂部變數。
24. **純原子類開發 (No Inline Styles)**：嚴禁使用 `style="..."`。所有樣式必須通過 Tailwind CSS 原子類實現。
25.  **語義化標籤**：導航區用 `nav`，主體用 `main`，統計區用 `section`，確保 SEO 與代碼結構清晰。
26. **資產路徑穩健性**：所有本地資源（圖片、JSON）必須使用相對路徑 (`./assets/...`)。
27. **高效 DOM 操作**：在 JS 渲染大量數據時，應使用 `template strings` 分塊渲染，或優化 DOM 更新次數。
28. **自動化註釋系統**：AI 產出的複雜 UI 邏輯必須自帶 JSDoc 註釋，說明輸入參數與渲染預期。

### 6. 專業增強功能 (Advanced Features)
29. **數據簽章與防偽**：匯出關鍵報告時，必須自動嵌入 Checksum 字串，並在 UI 下方顯示驗證標籤。
30. **隱藏捲軸設計 (Clean Scroll)**：所有內部滾動容器應移除原生捲軸外觀 (`no-scrollbar`)，改用微型自定義條或完全隱藏。
