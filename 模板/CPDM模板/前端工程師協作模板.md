# 前端工程師 Antigravity 協作模板

> 用於前端工程師在 Antigravity 中開發時的標準對話與工作模板

---

## 🚀 啟動開發對話

**複製並填寫以下內容作為與 Antigravity 的第一次對話:**

```markdown
# 前端開發任務啟動

## 專案上下文
- **專案名稱**: [例如: 企業請購系統]
- **我的角色**: 前端工程師 (Engineer A)
- **技術棧**: 
  - 框架: [例如: React 18 + TypeScript]
  - UI 庫: [例如: Tailwind CSS + ShadcnUI]
  - 狀態管理: [例如: Zustand]
  - API 客戶端: [例如: Axios]

## 參考文檔
- **架構設計**: [文檔連結]
- **API 規格**: [文檔連結]
- **Canvas 設計**: [文檔連結/截圖]

## 當前任務 (Task)
**任務名稱**: [例如: 實作一般請購單輸入畫面]
**功能描述**: 
1. 建立 PurchaseOrderForm 元件
2. 實作表單驗證 (Zod)
3. 串接 POST /api/orders 接口
4. 處理 Loading 與 Error 狀態

## 請求
請幫我分析上述任務，並根據 Canvas 設計與 API 規格，提供詳細的實作步驟與初步的程式碼架構。
```

## 🛠️ 開發過程中的常用 Prompt

### 1. 生成元件程式碼
> "請根據以下 Canvas 設計的欄位規格 (附貼上規格)，使用 React Hook Form + Zod 生成表單元件程式碼。請確保包含錯誤訊息顯示，並遵循專案的 UI 風格。"

### 2. API 串接與型別定義
> "這是後端提供的 API Response 範例 JSON (附貼上 JSON)。請幫我生成對應的 TypeScript Interface，以及一個使用 Axios 的 API 呼叫函式，包含錯誤處理。"

### 3. 解難與除錯 (Debug)
> "我在實作時遇到了這個錯誤 (貼上 Error Log)。相關的程式碼片段如下 (貼上程式碼)。請幫我分析原因並提供修正建議。"

### 4. 程式碼優化 (Refactor)
> "這個元件目前的邏輯有點複雜 (貼上程式碼)。請幫我重構它，將商業邏輯抽離到 Custom Hook 中，並提升程式碼的可讀性。"
