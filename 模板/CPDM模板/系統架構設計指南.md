## 4. API 設計規範

### 4.1 介面標準
- 路徑命名: `kebab-case` (例如 `/api/user-profiles`)
- 認證方式: Bearer Token (JWT)
- 分頁格式: `?page=1&limit=20`

### 4.2 核心 API 清單
| Method | Endpoint | 描述 | 備註 |
|--------|----------|------|------|
| GET | /api/resource | 取得列表 | 支援篩選 |

## 5. 非功能性需求 (NFRs)

### 5.1 效能 (Performance)
- API 回應時間 < 200ms

- 資料傳輸強制 HTTPS
- SQL Injection 防護 (使用 ORM)
- XSS/CSRF 防護

### 5.3 可靠性與備份
- 資料庫每日自動備份
- 服務健康檢查 (Health Check)
